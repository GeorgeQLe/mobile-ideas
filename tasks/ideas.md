# Mobile App Clone Ideas Backlog

Created: 2026-04-16

This backlog defines 100 mobile app clone projects for research and specification. "Clone" means functional parity research: reproduce core user flows, information architecture, interaction models, data contracts, and edge cases with original code, original branding, original copy, original media, and legally obtained data. Do not copy proprietary source code, private APIs, trademarks, copyrighted assets, or paywalled content.

## Research And Spec Standard

Each project should produce a spec before implementation:

1. Map the real app's core jobs-to-be-done, primary tabs, onboarding, account model, empty states, settings, notification flows, monetization, and offline behavior.
2. Capture screen-by-screen requirements from lawful observation: public app listings, first-party help docs, hands-on use, public screenshots, terms, privacy labels, accessibility behavior, and user reviews.
3. Define clone scope as functional parity, not brand parity: original name, visual system, copy, sample data, icons, and media.
4. Specify data models, API contracts, local cache rules, analytics events, permission prompts, push notification types, abuse controls, and test cases.
5. Include a "must not copy" section covering assets, trademarks, third-party content, proprietary algorithms, and private integrations.
