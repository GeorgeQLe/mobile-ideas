## Overview

Build an original mobile AI companion inspired by Pi's public product: an emotionally intelligent, voice-first conversational AI that prioritizes warmth, empathy, and personal memory over task completion. The clone focuses on natural conversation flow, proactive check-ins, emotional awareness, and a minimal distraction-free interface centered on a single continuous thread.

The clone must use original branding, UI copy, voice assets, and licensed model providers. Any feature marked as a manual verification blocker must stay behind a feature flag until lawful hands-on evidence confirms exact native behavior.

## Goals

- Provide a mobile-first AI companion with warm, empathetic conversational tone and emotional intelligence.
- Support voice-first interaction with natural speech input and expressive TTS output.
- Maintain personal memory across conversations so the companion recalls user context, preferences, and past topics.
- Deliver a minimal, distraction-free UI that keeps the focus on the conversation thread.
- Offer proactive check-ins and topic suggestions that feel natural rather than intrusive.
- Surface discoverable topics and conversation starters for users who want guided exploration.
- Ensure safety guardrails for sensitive emotional content including crisis detection and resource referral.

## Non-Goals

- Do not imply affiliation with Inflection AI, Pi, or Microsoft.
- Do not copy proprietary model weights, fine-tuning data, voice recordings, system prompts, or private API schemas.
- Do not provide medical, psychological, or crisis counseling; refer users to qualified resources.
- Do not position the app as a replacement for professional mental health support.
- Do not store raw voice recordings beyond the duration needed for transcription.
- Do not enable unsupervised use by minors without parental controls.

## Detailed Design

### Empathy-First Conversation Engine

- Every response is generated with an emotional-tone classifier that evaluates user sentiment before composing a reply.
- The model receives an emotional context object (detected mood, conversation arc, recent topic sensitivity) alongside the user message.
- Responses default to a warm, supportive, curious tone; the system never lectures, dismisses, or responds with clinical detachment.
- When the user expresses distress, the model shifts to active-listening mode: shorter responses, reflective statements, validation before suggestions.

### Voice-First Interaction

- Voice mode uses streaming speech-to-text for real-time transcription displayed as the user speaks.
- TTS output uses a warm, natural-sounding voice with prosody tuned for conversational cadence (not robotic or newscaster style).
- Users can interrupt TTS playback mid-sentence; the system stops speaking and listens immediately.
- Voice sessions track duration, turn count, and average response latency for quality monitoring.

### Personal Memory

- The system maintains a structured memory store per user: stated facts (name, job, interests), inferred preferences, and conversation summaries.
- Memory is updated incrementally after each conversation turn via an extraction pipeline.
- Users can view, edit, and delete individual memory entries from Profile and Preferences.
- Memory context is injected into the system prompt with a rolling window to stay within token limits.

### Proactive Check-Ins

- The system may send push notifications with conversational openers based on time-of-day patterns and recent topics.
- Notification frequency respects user-configured limits (default: at most once per day).
- Check-in content is generated from the memory store and topic suggestion engine, not canned templates.

## Core User Journeys

1. **First-time onboarding**: User installs, creates account, sees a brief warm welcome, chooses a display name, and enters their first conversation with a guided icebreaker prompt.
2. **Text conversation**: User opens the app, sees the continuous thread, types a message, receives a streamed empathetic response, and continues the exchange.
3. **Voice conversation**: User taps the microphone, speaks naturally, sees live transcription, hears a TTS reply, and can interrupt or continue hands-free.
4. **Topic discovery**: User visits Discover, browses suggested topics (e.g., "Tell me something surprising," "Help me think through a decision"), taps one, and it seeds a new conversational thread.
5. **Proactive re-engagement**: User receives a check-in notification, taps it, lands in the conversation with a personalized opener, and resumes chatting.
6. **Memory review**: User opens Profile, views what the companion remembers, edits or deletes entries, and confirms changes.

## Realtime Push And Offline Behavior

- SSE streaming for message responses; client renders tokens incrementally with typing animation.
- Push notifications for proactive check-ins delivered via APNs/FCM; payload includes a preview snippet and deep-link to conversation.
- Offline: drafted messages are queued locally and sent when connectivity returns. Previous messages are cached for read-only viewing.
- Voice mode requires an active network connection; offline voice attempts show a clear connectivity error.
- Background token refresh ensures sessions survive app backgrounding for up to 7 days.

## Permissions Privacy And Safety

- Microphone permission requested only when the user first taps the voice button; clearly explained rationale dialog.
- Notification permission requested after the user completes at least one conversation, not at first launch.
- No location, contacts, camera, or health data access required.
- Voice audio is streamed for transcription and not persisted beyond the request lifecycle.
- Memory entries are encrypted at rest and deletable by the user at any time.
- Crisis detection: if the model detects self-harm or crisis language, it surfaces verified crisis resources (988 Lifeline, Crisis Text Line) and logs a SafetyReport.
- All conversations are subject to content moderation; flagged content triggers review and optional user-facing warnings.
- COPPA: users must confirm age >= 13 at signup; under-13 accounts are blocked.
- Data export and account deletion comply with GDPR/CCPA requirements within 30 days.

## Analytics And Monetization

- Free tier: unlimited text messages, limited voice minutes per day (e.g., 15 minutes), basic memory.
- Premium tier: unlimited voice, extended memory depth, priority response speed, custom companion personality options.
- Subscription via Apple IAP and Google Play Billing; receipt validation server-side.
- Analytics events: session_start, message_sent, voice_session_start, voice_session_end, topic_tapped, memory_viewed, notification_opened, subscription_started, subscription_cancelled.
- Retention metrics: DAU/MAU, average session duration, voice vs. text ratio, notification open rate, memory edit frequency.

## Edge Cases

- **Voice recognition failure**: If STT confidence is below threshold, display a "I didn't catch that" message with a retry prompt rather than guessing.
- **Emotional context misreading**: If the sentiment classifier flags high distress but the user is discussing fiction or hypotheticals, include a gentle clarifying check ("Just checking in -- are you talking about something you're going through?").
- **Long conversation memory limits**: When the conversation exceeds the context window, summarize older turns and inject the summary; notify the user if very old details may be approximate.
- **Proactive notification timing**: Respect quiet hours strictly; if a check-in is suppressed, do not queue it for later -- generate a fresh one at the next eligible window.
- **Voice mode interruptions**: Phone calls, Siri activation, or audio focus loss must gracefully pause the voice session and allow seamless resumption.
- **Network loss during streaming**: If SSE connection drops mid-response, the client retries with a `Last-Event-ID` header; the server replays missed tokens.
- **Memory conflicts**: If the user corrects a memory entry that contradicts stored facts, the correction takes precedence and the old entry is archived, not deleted.
- **Subscription lapse**: Downgrade to free tier gracefully; do not delete premium memory -- make it read-only until the user re-subscribes.
