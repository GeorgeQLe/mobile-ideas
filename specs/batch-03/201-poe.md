## Overview

Build an original mobile product inspired by Poe's public workflow: multi-model AI chat through a unified interface, bot marketplace discovery, custom bot creation with system prompts and knowledge bases, creator revenue sharing, conversation threading per bot, and cross-model comparison. The clone must preserve the functional interaction model users expect while using original product language, independent model routing, original bot marketplace logic, and jurisdiction-aware compliance.

The clone must not copy Poe branding, screenshots, marketing copy, protected UI artwork, bot marketplace data, creator earnings data, private APIs, proprietary routing logic, or model provider agreements. Any feature marked manual verification required must ship behind a feature flag or acceptance-test blocker until lawful hands-on verification confirms behavior.

## Goals

- Provide a mobile-first multi-model AI chat platform with bot discovery, conversation management, bot creation tools, knowledge base uploads, subscription/compute management, and creator analytics.
- Replace discovery placeholders with exact public sources and translate those sources into implementation-ready product, data, API, safety, analytics, and test requirements.
- Preserve category trust expectations around model accuracy, response streaming, creator intellectual property, content safety, billing transparency, and privacy rights.
- Produce concrete screens, entities, API contracts, realtime/offline behavior, analytics events, edge cases, acceptance tests, and build phases for a downstream implementation repo.

## Non-Goals

- Do not build a Poe-branded app or imply affiliation with Poe, Quora, or any model provider.
- Do not scrape Poe, reuse private APIs, replay network traffic, copy marketplace bot data, copy creator content, clone proprietary routing/ranking systems, or reproduce policy copy.
- Do not process production payments, creator payouts, compute credits, identity documents, or subscription disputes without separate legal, compliance, privacy, and provider review.
- Do not claim exact App Store, Play Store, native-device, checkout, account, notification, creator-payout, or regional parity until manual verification blockers are resolved.
- Do not build runtime application code in this repository; this repo remains the planning and specification source.

## Detailed Design

### Source-Backed Product Requirements

- The public listings position Poe as a multi-model AI chat platform where users access GPT-4, Claude, Gemini, Llama, and other models through a single interface with per-bot conversation threads.
- The V1 must separate model routing, bot definitions, conversation state, knowledge bases, creator profiles, compute budgets, and subscription state into explicit data lifecycles.
- Bot creation must support system prompts, knowledge base attachment, API server integrations, suggested replies, intro messages, and visibility/pricing controls.
- The marketplace must surface trending, featured, category-filtered, and search-ranked bots with usage metrics and creator attribution.
- Cross-model comparison must allow side-by-side responses from multiple models to the same prompt within a single view.
- Compute credits must gate expensive model calls with clear budget visibility, top-up paths, and subscription-tier differentiation.
- Content safety must address prompt injection, jailbreak attempts, NSFW generation, harassment via bots, impersonation, and minors accessing age-inappropriate bots.
- Manual verification remains required for exact streaming token delivery, bot creator revenue dashboards, knowledge base indexing latency, compute metering accuracy, and subscription restore flows.

### Product Architecture Notes

- Use original navigation labels and component names while preserving the documented user jobs.
- Keep all user-generated content, AI outputs, bot configurations, knowledge base files, and payment state under explicit data ownership and deletion rules.
- Model every LLM provider as an adapter with feature flags, timeout behavior, token-counting, provider-specific error mapping, and cost tracking.
- Keep entitlement and compute-credit checks server-authoritative; the client may cache balance summaries but cannot grant paid model access alone.
- Store only normalized event codes in analytics; raw prompts, responses, knowledge base contents, and payment details are disallowed in logs.
- Provide creator tooling with analytics, earnings visibility, bot management, and abuse/takedown handling.
- Include localization, accessibility, age-gate, and regional availability states in route and API schemas.

## Core User Journeys

- New user installs the app, reviews onboarding, creates or restores an account, and reaches the bot discovery surface.
- User browses or searches the marketplace, selects a bot, reads its description and capabilities, and starts a conversation.
- User sends a message to a bot and receives a streamed response with typing indicators and token-level delivery.
- User compares responses from multiple models to the same prompt using the cross-model comparison surface.
- Creator builds a custom bot with system prompt, knowledge base, and pricing, publishes it, and monitors analytics.
- User uploads documents to a knowledge base, attaches it to a bot, and queries the bot with retrieval-augmented responses.
- User hits compute credit limits, sees clear budget feedback, and upgrades subscription or purchases additional credits.
- User reports an unsafe bot or response, blocks the creator, and receives safety feedback with support escalation path.
- User requests data export or account deletion and sees confirmation, retention timing, and final status.

## Data Model

- `User`: identity, email, OAuth providers, locale, region, account status, consent state, deletion/export state.
- `DeviceSession`: platform, app version, auth token state, notification token, last active timestamp.
- `Bot`: name, description, system prompt, intro message, suggested replies, visibility, pricing, creator reference, model provider, category tags, usage count.
- `BotCreatorProfile`: user reference, display name, bio, earnings balance, payout method, verification status, published bot count.
- `Conversation`: user reference, bot reference, title, created/updated timestamps, message count, pinned state, archived state.
- `Message`: conversation reference, role (user/assistant/system), content parts, token count, model used, created timestamp, feedback state.
- `ContentPart`: message reference, type (text/image/file/code), body or media reference, rendering hints.
- `KnowledgeBase`: creator reference, name, document list, indexing status, chunk count, last updated, attached bot references.
- `Subscription`: user reference, tier, platform owner, status, period start/end, compute credit balance, auto-renew state.
- `ModelProvider`: identifier, display name, capabilities, token limits, cost per token, availability status, feature flags.
- `SafetyReport`: reporter, target bot/message/creator, category, evidence, status, moderator notes, resolution.
- `AuditEvent`: append-only sensitive account, privacy, billing, moderation, safety, and creator-payout changes.

## API And Backend Contracts

- `POST /auth/session`, `POST /auth/oauth`, `DELETE /auth/session`: device-scoped authentication with OAuth providers and abuse throttles.
- `GET /bots?cursor=&category=&sort=`, `GET /bots/:id`, `POST /bots`, `PATCH /bots/:id`, `DELETE /bots/:id`: bot CRUD with marketplace discovery, filtering, and creator authorization.
- `GET /conversations?cursor=&bot=`, `POST /conversations`, `DELETE /conversations/:id`: conversation lifecycle with bot-scoped threads and bulk deletion.
- `POST /conversations/:id/messages` (SSE streaming response), `GET /conversations/:id/messages?cursor=`: message send with server-sent event streaming for token delivery, pagination for history.
- `POST /uploads`, `PUT /uploads/:id/content`, `POST /uploads/:id/complete`: signed upload flow for attachments and knowledge base documents with MIME/size validation.
- `POST /knowledge-bases`, `GET /knowledge-bases/:id`, `PATCH /knowledge-bases/:id`, `DELETE /knowledge-bases/:id`, `POST /knowledge-bases/:id/documents`: knowledge base CRUD and document indexing lifecycle.
- `GET /creator/analytics`, `GET /creator/earnings`, `POST /creator/payout-method`: creator dashboard data, earnings history, and payout configuration.
- `GET /subscriptions/current`, `POST /checkout/session`, `POST /billing/restore`, `POST /billing/webhook`, `POST /credits/purchase`: subscription and compute credit management with platform-specific restore.
- `POST /search`, `GET /search/suggestions`: global search across bots, conversations, and messages with filters and safe-mode.
- `POST /reports`, `POST /blocks`: trust-and-safety lifecycle with category, evidence, and moderator audit trails.
- `GET /notifications`, `PATCH /notification-preferences`: notification delivery and preference management.

## Realtime, Push, And Offline Behavior

- Stream model responses via SSE with token-level granularity, typing indicators, and graceful timeout/reconnect handling.
- Cache the bot discovery surface, recent conversations, settings, and subscription state for offline reads.
- Queue user messages locally when offline with retry metadata; block irreversible actions (deletion, purchases, payouts) while offline.
- Realtime updates must reconcile against server state after reconnect to avoid duplicate messages or stale conversation state.
- Push notifications must be opt-in, category-scoped (new responses, bot updates, credit alerts, creator earnings), and free of raw message content by default.
- Background work must tolerate app termination, token expiry, OS permission revocation, and provider timeouts.
- Deleted, suspended, or blocked bots must be removed from caches and active conversation lists consistently.

## Permissions, Privacy, And Safety

- Treat prompt injection and jailbreak attempts as a launch-blocking review area with detection, mitigation, and audit coverage.
- Treat NSFW/harmful content generation as a launch-blocking review area with per-model safety layers and user reporting.
- Treat minor access to age-inappropriate bots as a launch-blocking review area with age-gate and content classification.
- Treat creator intellectual property (system prompts, knowledge bases) as confidential with access controls and theft prevention.
- Treat compute credit fraud and billing manipulation as a launch-blocking review area with server-authoritative metering.
- Request camera, photos, files, and notifications only when the user invokes a feature that needs them.
- Provide permission-denied fallbacks, settings education, no dark patterns, and no silent re-prompts.
- Provide user-visible privacy policy, terms, data export, delete account, report abuse, block, and support escalation paths.
- Minimize sensitive data in analytics, crash reports, and model provider logs; never log raw prompts or responses in analytics pipelines.
- Include age, region, regulated content, and accessibility review before launch.

## Analytics And Monetization

- Track privacy-safe lifecycle events: onboarding completed, bot discovered, conversation started, message sent, comparison used, bot created, knowledge base uploaded, subscription changed, credit purchased, report submitted.
- Every event must use stable object types and failure codes, not raw prompts, responses, or private identifiers.
- Monetization tiers: free (limited daily messages and model access), subscriber (higher limits, premium models, priority), creator earnings (revenue share on per-message compute from users chatting with paid bots).
- Compute credit system must expose balance, usage rate, model-specific costs, and top-up paths transparently.
- Creator revenue sharing must track per-bot message volume, compute cost attribution, earnings accrual, and payout schedules.
- Billing tests must cover free, subscriber, expired, canceled, refunded, restored, credit-purchased, and creator-payout states.

## Edge Cases

- First launch offline, unsupported OS, blocked region, underage user, expired session, suspended account, or service outage.
- Model provider outage mid-stream, partial response delivery, timeout after partial tokens, and provider-specific error codes.
- Compute credits exhausted mid-conversation, subscription lapse during active chat, and concurrent sessions draining shared balance.
- Bot deleted or suspended while user has active conversation, creator account banned with published bots still cached.
- Knowledge base indexing failure, oversized document upload, unsupported file format, and corrupted index state.
- Duplicate message sends from retry, stale optimistic UI, SSE reconnect delivering duplicate tokens, and app termination during stream.
- Creator disputes: earnings calculation disagreement, payout failure, bot plagiarism claims, and takedown appeals.
- Abuse: prompt injection via bot system prompts, jailbreak sharing in marketplace, impersonation bots, spam bot publishing, and coordinated manipulation.
