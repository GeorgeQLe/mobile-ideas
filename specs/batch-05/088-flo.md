# 088 Flo Clone Spec

## Legal Scope
- Clone cycle tracking, period predictions, symptom logging, and educational content.
- Use original branding and editorial content.

## Product Goal
- Help users track cycles, understand likely fertile windows, and log related symptoms privately.

## Research Verification Checklist
- Public listing, onboarding, and content gating: verify.
- Prediction behavior, reminders, and privacy defaults: verify.
- Partner/share flows and premium features: verify.

## Core User Journeys
- User enters period dates and receives cycle predictions.
- User logs symptoms, mood, and notes daily.
- User views fertility and period calendars.
- User upgrades for richer insights and content.

## Screen Inventory
| Screen | Purpose | Inputs | States | Edge Cases |
|---|---|---|---|---|
| Home | Daily check-in | Symptoms, note | Empty, active | No cycle data |
| Calendar | Predictions | Cycle dates | Month view, detail | Irregular cycle |
| Logging | Daily entry | Mood, symptom, flow | Draft, saved | Missing date |
| Insights | Pattern view | Time range | Weekly, monthly | Sparse history |
| Paywall | Premium | Plan, restore | Trial, error | Cancelled |

## Functional Requirements
- Track periods, cycle length, symptom logs, mood, and notes.
- Generate predicted period and fertile windows from stored history.
- Allow daily reminders and check-ins.
- Support optional anonymous educational content consumption.
- Keep records private by default.

## Data Model
- `User`, `Cycle`, `PeriodEntry`, `SymptomLog`, `Prediction`, `Reminder`, `ContentBookmark`, `Subscription`.
- Separate sensitive health data from generic user profile data.

## API/Backend Contracts
- `POST /cycles`
- `POST /symptoms`
- `GET /calendar`
- `GET /insights`
- `POST /reminders`

## Realtime/Push/Offline
- Push for reminder nudges and predicted window alerts.
- Offline entry creation with later sync.
- No realtime social layer required.

## Permissions/Privacy/Safety
- Strong privacy defaults and account lock options.
- Make health predictions clearly informational, not diagnostic.
- Support passcode or biometric app lock if platform allows.

## Analytics Events
- `cycle_started`, `symptom_logged`, `prediction_viewed`, `reminder_set`, `content_opened`, `note_added`, `paywall_viewed`, `app_locked`.

## Monetization
- Freemium with premium insights, predictions history, and content library access.

## Acceptance Tests
- Add a new period date and see predictions update.
- Log symptoms offline and sync later.
- Lock the app and verify private data access is blocked.
- Restore purchase and unlock premium insights.

## Implementation Notes
- Treat all dates in the user timezone to avoid cycle drift.
- Keep prediction outputs explainable and easy to override.
- Minimize retention of sensitive logs where possible.
