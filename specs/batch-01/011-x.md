# X-Style Microblog Clone Spec

## Legal Scope
- Clone objective: create a real-time text-first social app with posting, replies, reposts, quote posts, topics, lists, and audio rooms.
- Must not copy: X branding, logos, ranking code, private moderation tooling, or proprietary subscription names.
- Replacement brand/assets: use original app name, icon set, and UI copy.

## Product Goal
- Let a user post short updates, follow a live feed, join conversations, and monitor trends quickly from mobile.

## Research Verification Checklist
- [ ] Home timeline ranking and tabs
- [ ] Post, reply, repost, and quote-post composition
- [ ] Topics, lists, and search/discovery behavior
- [ ] Audio room start/join/moderation flow
- [ ] Premium subscription gates and verification labels
- [ ] Notifications, mentions, and inbox behavior
- [ ] Blocking, muting, and abuse reporting
- [ ] Offline drafts and reconnect behavior

## Core User Journeys
- User posts a text update and receives replies.
- User opens a live topic feed and follows a conversation.
- User adds another account to a list and revisits it later.
- User joins an audio room and returns to the timeline.
- User edits privacy, blocked words, and notification settings.

## Screen Inventory
| Screen | Purpose | Key States |
|---|---|---|
| Home Timeline | Browse live feed | following, ranked, empty |
| Compose | Write a post | draft, warning, posted |
| Post Detail | View thread and metrics | replies, reposts, locked |
| Search/Trends | Discover topics | trending, filtered, no results |
| Lists | Curated account groups | empty, saved, shared |
| Audio Room | Join voice discussion | waiting, live, muted |
| Notifications | Mentions and follows | unread, cleared |
| Settings | Privacy and safety | default, restricted |

## Functional Requirements
- Support text posts, media attachments, reply chains, reposts, and quote-posts.
- Provide a timeline with latest and ranked modes plus search and trend views.
- Support lists, bookmarks, and topic tags.
- Provide live audio rooms with hosts, speakers, listeners, and moderation controls.
- Support verified/badge-like account states through an internal trust system.
- Preserve drafts and allow post editing with revision history where supported.

## Data Model
- User: id, handle, display_name, bio, trust_status, privacy.
- Post: id, user_id, body, media_refs, reply_to_id, repost_of_id, quote_of_id.
- List: id, owner_id, name, description, member_ids.
- Topic: id, slug, title, trend_score.
- AudioRoom: id, host_id, title, status, participant_count.
- Notification: id, user_id, type, payload, read_at.

## API/Backend Contracts
- `GET /timeline?mode=following|ranked`
- `POST /posts`, `PATCH /posts/:id`, `POST /posts/:id/replies`
- `POST /posts/:id/reposts`, `POST /posts/:id/quote`
- `GET /topics`, `GET /search?q=`, `GET /lists/:id`
- `POST /audio-rooms`, `POST /audio-rooms/:id/join`, `PATCH /audio-rooms/:id`
- Provide websocket/SSE for live room and notification updates.

## Realtime/Push/Offline
- Push on mentions, replies, reposts, and live room starts.
- Stream timeline updates and room events in realtime.
- Cache drafts, timelines, and list memberships offline.
- Retry compose and media actions when network returns.

## Permissions/Privacy/Safety
- Support protected accounts, blocked words, and conversation controls.
- Apply rate limits and abuse checks to replies and reposts.
- Use strict moderation for live audio rooms and host controls.
- Keep sensitive analytics out of raw post bodies when possible.

## Analytics Events
- `timeline_opened`, `post_created`, `reply_sent`, `repost_created`
- `quote_created`, `list_saved`, `audio_room_joined`, `search_used`
- `mention_opened`, `notification_read`, `user_blocked`

## Monetization
- Ad-supported timeline plus optional subscription tiers for premium account features.
- Premium may unlock enhanced analytics, long posts, or stronger profile customization.

## Acceptance Tests
- User can post, reply, repost, and quote without losing drafts.
- User can join a live audio room and return to the feed.
- User can create a list and revisit it after restart.
- Protected account content stays hidden from non-followers.
- Offline draft survives app close and posts after reconnect.

## Implementation Notes
- Keep timeline generation isolated from notification and posting services.
- Treat audio rooms as stateful sessions with strict moderation and timeout rules.
- Separate rankable content from user privacy settings for predictable filtering.
- Verify exact tabs, subscription gates, and room controls with live research.
