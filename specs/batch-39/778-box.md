## Overview

Build an original mobile-first enterprise cloud content management platform inspired by Box's public user-facing workflow. The clone delivers secure file storage, collaboration, workflow automation, metadata-driven content governance, electronic signatures, and enterprise-grade access control. Core focus areas: file/folder operations with versioning, sharing and external collaboration with granular permissions, sync and offline with conflict resolution, security policies (encryption, DLP, retention, legal hold), storage quota management, Box Drive-style desktop sync, Box Notes collaborative editing, and Box Sign document workflows.

## Goals

- Deliver an enterprise-grade cloud content management experience with file storage, preview, collaboration, workflows, metadata, retention, and compliance controls.

## Detailed Design

### File Operations

- **Upload**: Support single-file, multi-file, and folder upload via file picker, drag-and-drop (web/desktop), camera capture, and share-extension intake. Enforce per-file size limits (5 GB free, 15 GB enterprise), MIME validation, malware scanning, and duplicate-name resolution (auto-rename or version).
- **Download**: Stream files with range-request support for resume. Respect download permissions per collaboration role. Track download events in audit log.
- **Preview**: Render 120+ file types in-app using server-side conversion (PDF, Office, images, video, audio, CAD, DICOM). Support page navigation, zoom, annotation overlay, and watermark injection for restricted content.
- **Edit**: Launch native editor integration for Office files (via WOPI-style protocol), rich-text notes (Box Notes equivalent), and plain-text editing. Lock file during edit session with configurable lock timeout.
- **Move/Copy**: Batch operations with progress indicator. Enforce destination folder permissions. Preserve or reset sharing permissions based on admin policy.
- **Delete/Restore**: Soft-delete to trash with configurable retention (30-day default, admin-configurable). Permanent purge requires elevated permission. Restore preserves version history and metadata.
- **Versioning**: Auto-version on upload of same-named file. Display version history with uploader, timestamp, size delta. Allow version promotion, download of prior versions, and admin-configurable version limits (50-100 per file).

### Sharing And Collaboration

- **Shared links**: Generate view-only, download-enabled, or edit-enabled links with optional password, expiration date, download count limit, and vanity URL slug.
- **Collaborator roles**: Owner, Co-Owner, Editor, Viewer Uploader, Previewer Uploader, Viewer, Previewer with granular permission matrix (upload, download, preview, edit, delete, share, manage).
- **External collaboration**: Invite external email addresses with configurable allow/block lists per domain. Require acceptance of terms. Show external badge on shared content.
- **Collaboration groups**: Create managed groups for batch permission assignment. Support nested groups with inherited permissions.
- **Real-time co-editing**: Box Notes equivalent with operational-transform or CRDT-based concurrent editing, presence indicators, cursor tracking, and comment threads.
- **Watermarking**: Apply visible watermarks (user email + timestamp) to previews and downloads for classified content.
- **Access requests**: Non-collaborators can request access; owner receives notification and can approve/deny with role selection.

### Sync And Offline

- **Desktop sync (Box Drive equivalent)**: Virtual filesystem driver that streams files on-demand. Pin files/folders for offline availability. Show sync status badges (synced, syncing, online-only, error).
- **Mobile offline**: Mark files/folders for offline access. Background-download with Wi-Fi-only option. Automatic cache eviction when storage quota exceeded.
- **Conflict resolution**: Detect concurrent edits via ETag/version mismatch. Present user with keep-both, keep-local, or keep-server options. Auto-resolve for non-overlapping metadata changes.
- **Selective sync**: Choose which top-level folders sync to desktop. Exclude large media folders to conserve disk space.
- **Bandwidth management**: Admin-configurable upload/download throttling. Pause/resume sync. Metered-network detection with user-configurable policy.
- **Sync status reporting**: Per-file and per-folder sync state exposed via status tray, file-explorer overlay icons, and in-app sync activity feed.

### Security And Compliance

- **Encryption**: AES-256 at rest, TLS 1.3 in transit. Customer-managed encryption keys (CMEK) for enterprise tier via key management service integration.
- **Data Loss Prevention (DLP)**: Content-inspection policies that scan uploaded files for sensitive patterns (PII, PCI, PHI). Actions: alert, quarantine, restrict sharing, or block upload. Admin-configurable rule sets.
- **Retention policies**: Time-based retention with indefinite hold option. Prevent deletion of content under retention. Disposition actions: auto-delete, review-then-delete, or archive.
- **Legal hold**: Place users, folders, or content under litigation hold. Override retention disposition. Audit all hold assignments and releases.
- **Classification labels**: Apply sensitivity labels (Public, Internal, Confidential, Restricted) to files/folders. Labels drive DLP policy, watermarking, sharing restrictions, and download controls.
- **Access logging**: Immutable audit trail of all file events (view, download, edit, share, delete, permission change, login). Export to SIEM via webhook or scheduled report.
- **Device trust**: Require managed-device enrollment for mobile access. Block jailbroken/rooted devices. Enforce app PIN/biometric lock.

### Storage Management

- **Quotas**: Per-user and per-enterprise storage quotas. Visual usage indicator with breakdown by file type. Warn at 80% and 95% thresholds.
- **Cleanup recommendations**: Surface large files, old files, duplicate files, and trash contents for review. One-tap cleanup actions.
- **Archival**: Cold-storage tier for infrequently accessed content. Auto-archive based on last-access date. Restore from archive with configurable retrieval time.
- **Admin storage reports**: Per-user, per-department, per-content-type usage dashboards. Export CSV/PDF.

### Workflow Automation (Box Relay Equivalent)

- **Workflow templates**: Pre-built templates for document approval, content review, and onboarding packet assembly.
- **Custom workflows**: Visual workflow builder with trigger (upload, metadata change, date, manual), condition (file type, folder, metadata value, user role), and action (assign task, move file, apply metadata, send notification, change classification) nodes.
- **Task assignments**: Assign review/approval tasks to users or groups with due dates. Track completion status. Escalate overdue tasks.
- **Outcome tracking**: Dashboard showing active workflows, completion rates, bottlenecks, and SLA compliance.

### Electronic Signatures (Box Sign Equivalent)

- **Document preparation**: Upload PDF or convert Office document. Add signature fields, initials, date, text, and checkbox placeholders. Assign fields to signers by role.
- **Signing ceremony**: Email notification with secure link. In-app or browser signing with draw, type, or upload signature options. Mobile-optimized signing experience.
- **Audit trail**: Immutable signing log with IP, timestamp, and device for each action. Generate signed certificate of completion.
- **Status tracking**: Pending, viewed, signed, declined, expired states with reminder automation.

## Core User Journeys

- **First-time individual user**: Downloads app, creates account with email/SSO, completes storage-primer onboarding, uploads first file, previews it, creates a shared link, and sends to a colleague.
- **Enterprise user onboarding**: Receives invite from admin, authenticates via corporate SSO, device trust check passes, lands on pre-populated department folder, pins critical folder for offline access.
- **File collaboration cycle**: User uploads document to shared folder, collaborator receives notification, opens preview, adds comment, requests edit access, edits via integrated editor, saves new version, original uploader reviews version diff.
- **External sharing**: User creates shared link with download disabled and expiration, sends to external partner, partner previews with watermark, requests download permission, user upgrades link permissions with password protection.
- **Workflow execution**: Manager triggers approval workflow on contract, assigns legal reviewer, reviewer opens file in preview, adds comment requesting revision, workflow routes back to author, author uploads revised version, reviewer approves, workflow moves file to executed-contracts folder and applies retention policy.
- **Offline/sync recovery**: User pins folder for offline on mobile, edits a file while disconnected, reconnects, app detects version conflict, user chooses to keep both versions, conflict copy is created with timestamp suffix.
- **Compliance scenario**: Admin creates DLP policy for PCI data, user uploads file containing card numbers, system quarantines file, notifies admin and user, admin reviews and releases or permanently quarantines.
- **Storage quota management**: User approaches quota limit, receives warning notification, opens storage management screen, reviews large/old files, moves archive-eligible content to cold storage, deletes trash, confirms new usage level.

## Realtime, Push, And Offline Behavior

- **Server-Sent Events**: Long-poll or SSE endpoint for real-time file change notifications, collaboration invitations, task assignments, and workflow state changes.
- **WebSocket fallback**: For co-editing sessions (Box Notes equivalent), maintain WebSocket with OT/CRDT operation stream, presence heartbeat, and graceful reconnection with operation replay.
- **Push notifications**: FCM/APNs for file shared with you, comment/mention, task assigned/due, sign request received, quota warning, security alert. Opt-in per category.
- **Offline caching strategy**: Cache folder tree metadata (last 30 days accessed), pinned file content, recent previews, and pending upload queue. Total cache budget configurable (default 2 GB mobile, 50 GB desktop).
- **Queued operations**: Queue renames, moves, deletes, metadata edits, and comments while offline. Replay on reconnect with conflict detection per item ETag.
- **Blocked offline operations**: Block sharing permission changes, workflow triggers, sign requests, DLP overrides, retention changes, and permanent deletes while offline.
- **Reconnection protocol**: On reconnect, fetch delta events since last `stream_position`, reconcile local state, surface conflicts requiring user action, and resume background sync.
- **Background sync**: iOS background fetch + BGTaskScheduler; Android WorkManager with network constraint. Respect battery-saver and metered-network settings.

## Permissions, Privacy, And Safety

- **Data loss prevention**: Launch-blocking. Content scanning for PII/PCI/PHI patterns. Quarantine workflow for policy violations. Admin review queue.
- **Permission model**: Hierarchical folder permissions with inheritance. Explicit deny overrides inherited allow. External collaboration requires enterprise-level opt-in.
- **Encryption**: AES-256 at rest with per-file keys wrapped by enterprise master key. Optional CMEK integration. TLS 1.3 for all transit. Certificate pinning on mobile.
- **Device trust**: MDM enrollment verification, jailbreak/root detection, app-level PIN/biometric, remote wipe of cached content.
- **Audit immutability**: All audit events append-only with cryptographic hash chain. No admin can delete or modify audit records.
- **Data residency**: Region-specific storage with configurable data-residency zones (US, EU, AU, JP, CA). Cross-region transfer requires explicit consent.
- **Content moderation**: Malware scanning on upload. Optional image/video content safety scanning for enterprise compliance.
- **Privacy controls**: Granular consent for analytics, marketing communications, third-party integrations. Data export (GDPR Article 15) and deletion (Article 17) from settings.
- **Permission request timing**: Request camera only for document scan, photos only for upload, notifications only after first collaboration event. Provide fallback for all denials.

## Edge Cases

- **File conflicts**: Two users edit same file simultaneously without lock. Last-write-wins with automatic version creation for the overwritten state. Notification to both users.
- **Quota exceeded during upload**: Reject upload at API level with clear error. Resume-capable uploads preserve already-transferred chunks for 24 hours.
- **Shared link to deleted file**: Return 404 with "This content is no longer available" message. Do not reveal file name or owner.
- **Retention prevents deletion**: User attempts delete on retention-locked file. Show inline explanation of retention policy. Offer to request admin exemption.
- **Legal hold on user leaving**: Preserve all content under hold when user account is deactivated. Transfer non-held content to designated successor.
- **DLP quarantine during collaboration**: File quarantined after collaborator uploads sensitive content. Existing collaborators see "under review" placeholder. Admin can release or redact.
- **SSO provider outage**: Cache last-valid session for 4-hour grace period. Show banner explaining degraded auth. Block new device enrollments.
- **Sync with filesystem special characters**: Sanitize file names containing characters invalid on target OS. Maintain server-canonical name with local alias mapping.
- **Concurrent workflow triggers**: Deduplicate workflow executions when multiple triggers fire within debounce window. Idempotent action execution.
- **Sign request expired**: Auto-cancel after expiration. Notify sender. Allow sender to create new request from same prepared document.
- **Version limit reached**: When file exceeds max version count, auto-purge oldest non-retained versions. Notify file owner.
- **External collaborator domain blocked**: Admin adds domain to block list after existing collaborations exist. Existing collaborations enter 7-day grace period with warning, then auto-remove.
- **Offline edit on file that was deleted server-side**: On reconnect, detect deletion, prompt user to save as new file or discard local changes.
- **Preview conversion timeout**: Large/complex files may timeout during server-side conversion. Show progress indicator, retry once, then offer download as fallback.
