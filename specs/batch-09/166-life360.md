## Overview

Build an original mobile family-locator app inspired by Life360: create a family circle, share real-time location, receive place-based arrival/departure alerts, view driving reports with events like hard braking and speeding, trigger SOS alerts, and optionally summon roadside assistance. Strong emphasis on mutual consent and anti-stalking safeguards.

## Goals

- Create "circles" (families/groups) and share real-time location with members.
- Define places (home, school, work) and alert on arrivals/departures.
- Offer driving summaries with originally computed events.
- Provide SOS alerts to circle members with one-tap dispatch metadata.
- Respect consent: all members can see who is tracking them and leave at any time.

## Non-Goals

- Do not enable covert tracking; device owner must be visible and consenting.
- Do not share data with advertisers or third-party analytics for location content.
- Do not copy proprietary driving-event ML or brand features.
- Do not operate as an emergency dispatch service beyond alerting circle members.

## Detailed Design

### Source-Backed Product Requirements

- Account with age gate; minors require parental consent per region.
- Circle creation with invite code or link; members must accept.
- Live location shared while app has permission; visible to all members.
- Places with geofence arrival/departure alerts.
- Driving trip detection with events (hard braking, acceleration, phone use), original implementation.
- SOS alert broadcasting location and status to circle.
- Roadside assistance handoff to a licensed third-party, not directly operated.
- Anti-abuse: anyone can see who is tracking them and leave a circle without silent persistence.

## Core User Journeys

- User installs, creates a circle, invites family.
- Invitee installs, accepts invite, grants location permission, sees circle map.
- User defines home and work places; arrivals trigger push to members.
- User takes a drive; trip detected; summary appears after arrival.
- User triggers SOS; members receive alert with last known location.
- User leaves a circle; location sharing to that circle stops immediately.
- User checks "who can see me" list and removes a member or leaves.
- User upgrades to premium; sees original entitlement copy.
- User disables location; circle shows "location off" state.
- User deletes account; all location history purged.

## Data Model

- `User`: identity, phone/email, age gate, consent state.
- `Circle`: id, owner, members, invite codes.
- `Membership`: user, circle, role, joined-at, left-at.
- `LocationUpdate`: user, coordinates, accuracy, battery, timestamp, motion state.
- `Place`: circle, geofence shape/radius, name, members notified.
- `Alert`: type (arrival/departure/sos/driving), subject user, timestamp, details.
- `Trip`: user, start, end, path, events (original).
- `SOSEvent`: user, timestamp, location, status, acknowledged-by list.
- `RoadsideRequest`: user, handoff vendor id, status.
- `Entitlement`: plan, platform.
- `PrivacySettings`: retention window, analytics opt-in, crash/driving opt-in.
- `AuditEvent`: membership, location-share changes, SOS, privacy events.

## Realtime, Push, And Offline Behavior

- Location updates batched; upload frequency adapts to battery and motion.
- Websocket or push for real-time map and alerts.
- Offline queue for updates; replay on reconnect.
- Push notifications for alerts and SOS; SOS bypasses quiet hours with user consent.
- Background location requires explicit "Always" permission with clear disclosure.

## Permissions, Privacy, And Safety

- Precise location requires explicit "Always" permission; "While in use" fallback degrades gracefully.
- Anti-stalking: members see who tracks them and can leave unilaterally; no silent/covert mode.
- Minors require parental/guardian consent per region; COPPA-style review for under-13.
- Domestic-abuse threat model: cooldown before adding a new member, visible notification to existing members when someone joins, inability to hide from other members.
- Location history retention configurable and exportable; default minimal.
- No location data to advertisers.
- Driving-event analysis opt-in and revocable.
- Support tooling redacts location; raw access requires elevated approval.
- Launch owner: privacy lead, safety/trust lead, accessibility owner.

## Analytics And Monetization

- Track privacy-safe events: circle created, invite accepted, place created, SOS triggered, subscription events.
- No location content in analytics payloads.
- Free tier plus premium (extended history, roadside, more places); original paywall copy.
- Roadside dispatch handled by licensed partner; the app surfaces status only.

## Edge Cases

- User denies background location; app runs in limited mode.
- Battery saver delays updates; explain staleness.
- Device offline in remote area; queue and replay.
- SOS triggered accidentally; 10-second cancel window.
- Circle owner leaves; ownership transfers to next admin.
- Minor added without explicit consent; block until verified.
- Spoofed location detection.
- Trip boundary errors (e.g., false starts).
- Roadside request in unsupported region.
- Account deletion with active SOS.
